## What it does

Converts JPEG and PNG images into Markdown by layering two sources of information: EXIF metadata extracted by the external `exiftool` binary, and a natural-language description generated by an LLM vision API. Either layer can work independently ‚Äî you get EXIF without an LLM, and you get an LLM description without exiftool.

## Why it exists

Images are opaque to text-only LLMs. Without captioning, an image in a document is simply lost during conversion. By generating a textual description at conversion time, downstream models can reason about visual content even if they're text-only. EXIF metadata (photographer, GPS, date, keywords) adds structured context that the visual description might miss.

## How it actually works

**EXIF extraction:** If `exiftool` is installed on the system, the converter calls it as a subprocess via the shared `_exiftool.py` utility, requesting a specific set of fields: `ImageSize`, `Title`, `Caption`, `Description`, `Keywords`, `Artist`, `Author`, `DateTimeOriginal`, `CreateDate`, `GPSPosition`. The output is formatted as key-value pairs in the Markdown, e.g. `**Title**: Sunset over the bay`.

**LLM captioning:** If the MarkItDown instance was initialized with `llm_client` and `llm_model` parameters, the converter:
1. Reads the image bytes from the stream.
2. Base64-encodes them and constructs a `data:image/jpeg;base64,...` URI.
3. Calls the client's `chat.completions.create()` with a message containing both the text prompt (default: `"Write a detailed caption for this image"`) and the image as a `image_url` content part.
4. Appends the model's response text under a `## Description` heading.

The `llm_client` is expected to implement the OpenAI SDK interface ‚Äî any compatible client works (Anthropic, Azure OpenAI, local Ollama, etc.).

**Accepts check:** The converter only handles JPEG and PNG (by extension `.jpg`, `.jpeg`, `.png` or MIME `image/jpeg`, `image/png`). Other image types are declined.

## The non-obvious parts

- exiftool is an external binary, not a Python library. The converter invokes it via `subprocess` and parses its text output. If it's not on `PATH`, EXIF extraction is silently skipped.
- The `llm_prompt` is user-configurable at MarkItDown init time. You can change it to extract specific information: `"List any text visible in this image"`, `"Describe the chart's data"`, etc.
- PPTX conversion reuses this same captioning path for embedded slide images ‚Äî it's not duplicated.
- No local ML is involved for captioning. The heavy lifting is entirely in the API call.
